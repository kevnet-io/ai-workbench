# Security Policy

This is a public repository. All content is intended to be shareable.

## What NOT to Commit

- API keys or tokens
- Passwords or credentials
- Private/proprietary prompts containing customer or work data
- Personal identifiable information (PII)
- Any file matching patterns in `.gitignore` (secrets, credentials, env files)

## If You Find Leaked Secrets

If you discover accidentally committed secrets or sensitive data:

1. Open an issue immediately
2. Or contact the maintainer directly

Do not share or exploit the leaked information.

## Example Outputs

When including example model outputs, ensure they are:

- Synthetic or anonymized
- Free of any real personal data
- Safe to share publicly
